ISS_20180125 : ISS - IP 설정 문제로 발생하는 IP 주소 셋팅
- IP 방화벽 정책에 의해서 내부/외부 한개의 IP 셋팅만 가지고 프로그램을 실행 할 수 없음
	: IP 정책상에 내부/외부 를 모두 공개 할수 없음
	: 내부에서 접속 할 경우 외부 IP 설정으로 사용자가 서비스를 사용 할 수 없음
	: 외부 IP 문제로 사용자가 내부 IP로 접근 할 수 없음
- 해결 방법 1
	: 현재 접속된 사용자의 IP를 확인하여 Stream Base Url 주소를 hardcoding 으로 내부/외부 구분해서 변경 하는 기능이 필요함
	: 문제 1. 192.168.x.x 를 모두 내부 IP로 규정 할 경우, 외부의 공유기를 통해 접속하는 사용자의 IP도 192.168.x.x 가 될 수 있음.
		이럴 경우 외부의 공유기를 사용한 접근은 내부망으로 판단하게 되어, 외부방에서 접근 하였으나 내부망의 서비스를 이용 못함

- 해결 방법 2
	: 로그인시 내부 / 외부 구분으로 로그인을 하여 해당 설정에 대해 외부 / 내부 IP를 사용 할 수 있도록 가능함
	: 문제 1. 잘못 로그인 했을 경우 로그아웃이 귀찮음
	: 문제 2. 현재 내부에서 접속한 것인지 외부에서 접속한 것인지 화면상 판단 가능한 부분이 없음.
